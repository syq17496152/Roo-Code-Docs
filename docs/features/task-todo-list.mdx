# 任务待办清单

通过集成的待办管理功能，保持任务进度，帮助您在开发目标上保持有条理和专注。任务待办清单提供交互式、持久的检查清单，直接在聊天界面中跟踪您完成复杂、多步骤工作流程的进度。

:::tip 待办清单触发器
当处理复杂任务、多步骤工作流程或使用架构师模式时，会自动创建待办清单。您也可以通过要求 Roo "使用 update_todo_list 工具" 或 "创建待办清单" 来手动触发它们。

有关自动和手动触发的详细信息，请参阅 [Roo 创建待办清单的时机](#when-roo-creates-todo-lists)。

## 使用场景

**之前**：手动在脑海中或单独的笔记文件中跟踪复杂任务的步骤，很容易丢失进度和下一步骤。

**使用任务待办清单后**：Roo Code 会自动创建并管理嵌入在对话中的结构化检查清单。您可以查看每个项目的状态，观看 AI 标记项目完成，并在 Roo 提交更新以供批准时提供反馈。

## 它如何工作

任务待办清单功能由 [`update_todo_list`](/advanced-usage/available-tools/update-todo-list) 工具提供支持，该工具可在聊天界面中实现动态任务管理。

### Roo 创建待办清单的时机

Roo 通过自动检测和手动请求两种方式创建待办清单：

- **检测到任务复杂性** - 在您的请求中识别到多个步骤、阶段或依赖关系
- **在架构师模式下工作** - 始终创建待办清单作为结构化工作的主要规划工具
- **直接工具请求** - 说 "使用 update_todo_list 工具" 或 "请使用 update_todo_list"

请记住：即使手动触发，Roo 仍保持对待办清单内容和工作流程的控制。您在批准对话框中提供反馈，但 Roo 会根据任务需求管理清单。

### 显示和交互

待办清单出现在多个位置：

1. **任务标题摘要**：通过 `TodoListDisplay` 组件显示进度和下一个重要项目的精简只读显示

   <img src="/img/task-todo-list/task-todo-list-1.png" alt="显示待办清单进度的任务标题摘要" width="600" />

2. **交互式工具块**：通过聊天界面中的 `UpdateTodoListToolBlock` 组件，允许您：
   - 查看所有待办项目及其当前状态
   - 在 Roo 提交更新以供批准时编辑项目描述
   - 使用 "编辑" 按钮暂存更改（仅在 Roo 下次更新清单时应用）
   - 查看 Roo 管理待办工作流程时的进展

   <img src="/img/task-todo-list/task-todo-list-2.png" alt="聊天界面中的交互式待办清单工具块" width="600" />

3. **环境详情**：待办清单以 "REMINDERS" 表格的形式出现在环境详情中，为 AI 提供持久的待办状态访问权限

### 理解任务状态

Roo Code 会根据任务进度自动管理状态进展。每个待办项目有三种状态之一：

**待处理**：显示空复选框，表示任务尚未开始
![待处理的待办项目，显示空复选框](../../static/img/task-todo-list/task-todo-list-4.png)

**进行中**：显示黄色圆点指示器，表示任务正在处理中
![进行中的待办项目，显示黄色圆点指示器](../../static/img/task-todo-list/task-todo-list-5.png)

**已完成**：显示绿色对勾，确认任务已完成
![已完成的待办项目，显示绿色对勾](../../static/img/task-todo-list/task-todo-list-6.png)

## 自动批准待办清单更新

您可以启用自动批准待办清单更新，以减少长时间工作流程中的中断。启用后，Roo 将自动更新任务进度，无需每次更改都要求确认。

要配置此功能，请参阅 [待办清单更新自动批准设置](/features/auto-approving-actions#update-todo-list)。
