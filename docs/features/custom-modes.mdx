# 自定义模式

Roo Code 允许您创建 **自定义模式** 来定制 Roo 在特定任务或工作流中的行为。自定义模式可以是 **全局模式**（适用于所有项目）或 **项目专属模式**（仅在单一项目中定义）。

:::tip 粘性模型提升工作效率
每个模式——包括自定义模式——都具备 **粘性模型** 特性。这意味着 Roo Code 会自动记忆并选择您上次使用特定模式时使用的模型。当您切换模式时，Roo 将根据当前模式自动切换对应的首选模型，避免了频繁重新配置的麻烦。
:::

## 为何使用自定义模式？

*   **专业化定制：** 创建针对特定任务优化的模式，如“文档撰写者”、“测试工程师”或“重构专家”。
*   **安全性增强：** 限制某个模式对敏感文件或命令的访问权限。例如，“审查模式”可以被限制为仅读取操作。
*   **实验性尝试：** 可以在不影响其他模式的情况下安全地试验不同的提示词和配置方案。
*   **团队协作：** 与团队成员共享自定义模式以标准化工作流程。

## 自定义模式包含哪些内容？

自定义模式由多个关键属性定义。理解这些概念将有助于您有效定制 Roo 的行为。

| UI字段 / YAML属性                 | 概念性描述                                                                                                                                                                                                                                                                                                                                                                                       |
| ---------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 标识符 (`slug`)                   | 模式的 **唯一内部标识符**。该标识符会被 Roo Code 用于引用模式，特别是关联[模式专属指令文件](#通过文件目录指定模式专属指令)时使用。                                                                                                                                                                                                            |
| 名称 (`name`)                      | 模式在 Roo Code 用户界面中显示的 **展示名称**。该名称应当易于理解且具有描述性。                                                                                                                                                                                                                                                             |
| 描述 (`description`)               | 模式用途的 **简短、用户友好型摘要**，显示在模式选择器界面中。                                                                                                                                                                                                                                                                             |
| 角色定义 (`roleDefinition`)        | 定义模式核心身份和专长的内容。这部分文本将放置在系统提示词的开头部分。                                                                                                                                                                                                                                                                     |
| 可用工具组 (`groups`)              | 定义模式可访问的工具集和文件访问权限。                                                                                                                                                                                                                                                                                                     |
| 使用场景 (可选) (`whenToUse`)      | （可选）提供 Roo 自动决策的指导方针，特别是模式选择和任务编排方面。                                                                                                                                                                                                                                                                         |
| 自定义指令 (可选) (`customInstructions`) | 模式的 **具体行为指南** 或规则。这部分内容通常作为 `roleDefinition` 的补充说明添加到系统提示词的后半段。
