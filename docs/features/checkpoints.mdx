# 检查点

检查点功能会在 Roo Code 任务期间自动对工作区文件进行版本控制，使您能够安全地探索 AI 建议并轻松恢复不需要的更改。

检查点让您：
- 安全尝试 AI 建议的更改方案
- 轻松回滚到先前状态
- 对比不同的实现方法
- 在不丢失工作成果的前提下恢复到之前的项目状态

### Git 安装

检查点通过聊天界面直接集成到您的工作流中。

检查点会以两种形式出现在您的聊天历史中：

- **初始检查点** 标记了您项目的起始状态
   <img src="/img/checkpoints/checkpoints-1.png" alt="聊天中的初始检查点指示器" width="500" />

- **定期检查点** 在文件修改或命令执行后出现
   <img src="/img/checkpoints/checkpoints-2.png" alt="聊天中的定期检查点指示器" width="500" />

每个检查点提供两个主要功能：

### 查看差异

要比较当前工作区与某个检查点：

1. 在聊天历史中找到该检查点
2. 点击检查点的 `查看差异` 按钮

   <img src="/img/checkpoints/checkpoints-6.png" alt="查看差异按钮界面" width="100" />

3. 在比较视图中查看差异：
   - 新增行用绿色高亮显示
   - 删除行用红色高亮显示
   - 修改文件列出详细更改
   - 重命名和移动文件跟踪路径更改
   - 新增或删除的文件明确标记

### Restoring Checkpoints

To restore a project to a previous checkpoint state:

1. Locate the checkpoint in your chat history
2. Click the checkpoint's `Restore Checkpoint` button
   <img src="/img/checkpoints/checkpoints-7.png" alt="Restore checkpoint button interface" width="100" />
3. Choose one of these restoration options:
   
   <img src="/img/checkpoints/checkpoints-4.png" alt="Restore checkpoint option" width="300" />

   - **仅恢复文件** - 仅回滚工作区文件到检查点状态，不修改聊天记录。适合在保持聊天上下文的同时比较替代实现方案，允许您无缝切换不同的项目状态。此选项不需要确认，可让您快速切换不同的实现。

   - **恢复文件和任务** - 回滚工作区文件并删除后续聊天消息。当您想要完全重置代码和对话到检查点的时间点时使用。此选项需要确认对话框，因为无法撤销。

      <img src="/img/checkpoints/checkpoints-9.png" alt="恢复检查点文件和任务的确认对话框" width="300" />

### 限制和注意事项

- **范围**：检查点只捕获活动 Roo Code 任务期间的更改
- **外部更改**：任务之外的更改（手动编辑、其他工具）不包括在内
- **大文件**：非常大的二进制文件可能会影响性能
- **未保存的更改**：恢复会覆盖工作区中的任何未保存更改
